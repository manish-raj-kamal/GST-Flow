# Deployment

## Vercel environment variables

This Laravel app needs a few runtime variables in Vercel. Do not commit secret values to Git.

Set these in Vercel under Project Settings > Environment Variables for Production, Preview, and Development as needed:

```dotenv
APP_KEY=base64:...
MONGODB_URI=mongodb+srv://...
```

`APP_KEY` must be the same style of value generated by:

```bash
php artisan key:generate --show
```

The checked-in `vercel.json` already provides the non-secret defaults for the Vercel runtime, including `APP_ENV`, cache paths under `/tmp`, cookie sessions, and `DB_CONNECTION=mongodb`.

After adding or changing environment variables in Vercel, trigger a new deployment. Vercel only applies environment variable changes to new deployments.

## Quick production check

After redeploying, verify:

```bash
curl -I https://gst-flow-manager.vercel.app/
curl -I https://gst-flow-manager.vercel.app/up
```

Both should return a successful status. If they still return `500`, open the latest Vercel deployment logs and check first for missing `APP_KEY`, missing `MONGODB_URI`, or an unavailable MongoDB PHP extension in the selected PHP runtime.
